# Token visual review

**Status: DESIGN TOKEN REVIEW — NOT DEVICE VALIDATED.**

## The page
`reports/spatial-ux-v11/token-review/index.html` — offline, self-contained (all 37 semantic states
inlined from the canonical JSON at generation time), no CDN, no external request. Generated by
`scripts/generate-token-review.mjs` (deterministic; same bytes on re-run).

Open it directly (`file://`) — no server needed.

## What it shows
- **Distinctness pairs** at the top: the pairs that must never look the same
  (VERIFIED ≠ APPROVAL, VERIFIED ≠ REVIEW-RECORDED, FIRST-FAILURE ≠ DOWNSTREAM, WARNING ≠ SCANNING,
  SCANNING ≠ LOST, NOT-EVALUATED ≠ APPROVAL-ABSENT, SELF-SIGNED ≠ TIME-ANCHORED, FOCUSED ≠ SELECTED,
  BUILT ≠ DEVICE-VALIDATED) — each rendered as two coloured dots + labels so a reviewer can eyeball
  that they read as different states.
- **All 6 categories** (status, governance, trust posture, tracking, interaction, capability), each
  state as a card: colour swatch + EN name + ZH name + icon + shape + hex + EN a11y + ZH a11y.
- **Profile selector**: `DesktopDark` (semantic identity) and `GrayscaleReview`. Grayscale applies a
  CSS `grayscale(1)` filter to the whole grid — the review test is that **every state stays
  identifiable by shape + text with colour removed** (never colour alone).

## Header label
The page is banner-labelled **`DESIGN TOKEN REVIEW — NOT DEVICE VALIDATED`**. It reviews the design
token identities on a flat screen; it is not an OST/glasses capture and makes no device claim.

## How to regenerate
```
node scripts/generate-token-review.mjs
```
Re-run after any change to `design/shadow-spatial-tokens.json`.
