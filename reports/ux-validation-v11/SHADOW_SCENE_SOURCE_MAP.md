# Shadow Lens V11 — scene source map (offline, no Unity MCP)

Every path is repository-relative. Nothing here was learned from a filename alone: each entry was
opened and read.

## The two surfaces under audit

| Surface | Where it actually lives | Authority |
|---|---|---|
| **Audit Workspace** (Unity, the V11 product view) | `apps/shadow-lens/unity/Assets/ShadowLens/Workspace/ShadowAuditWorkspace.cs` builds the whole scene graph **at runtime**. There is **no committed `.unity` scene** for it — `Assets/ShadowLens/Scenes/ShadowV11BeamPro.unity` is generated by `Editor/ShadowV11BeamProCandidate.BuildScene()` at build time and is untracked. | `PROVEN_FROM_GENERATOR_CODE` |
| **Audit Room Flat** (Three.js, browser/WebXR) | `demos/replay/3d/` — `app.js` (composition + fit), `scene.js` (cards, inspector, leader line), `constants.js` (palette + geometry), `flat-fit.js` (pure fit math) | `PROVEN_FROM_GENERATOR_CODE` + browser-verified in an earlier increment |

**Consequence for this audit:** neither surface can be inspected as a serialized scene. Both are code.
That is also why Unity MCP would have added nothing here — the scene graph does not exist until the
code runs.
