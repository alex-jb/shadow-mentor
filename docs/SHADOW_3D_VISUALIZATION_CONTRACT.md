# Shadow 3D Visualization Contract — `shadow-3d-scene-v1`

One engine-neutral scene. **Unity and Three.js each ADAPT it** into their own objects. The contract
removes the current duplication where the tamper point, status vocabulary, and node sequence are
encoded three times (`demos/replay/3d/constants.js` `MUTATE_SEQ`, Unity `ShadowAuditChainData.BrokenAtSeq`,
`lib/claim-evidence-graph.mjs`). Schema: `schemas/shadow-3d-scene-v1.schema.json`. Fixtures:
`fixtures/shadow-3d/*.json` (generated by `fixtures/shadow-3d/build.mjs`). Tests:
`test/shadow-3d-scene-contract.test.js`.

## Authoritative vs advisory
- **Authoritative** (must be identical across engines): node/edge **IDs**, `type`, `sequence`, `status`,
  the `tamper` block (`failed_sequence` + `downstream_sequences`), the six-check `verification` +
  `analytical_correctness: NOT_EVALUATED`, `claim/evidence/source` IDs, `label_priority`, `accessibility`.
- **Advisory** (may differ per device/viewport): `layout_hint` (ring/angle/height/column). Positions are
  hints, never truth.
- **Forbidden in the contract**: GameObject / MonoBehaviour / Object3D names, engine materials, engine
  coordinates presented as truth.

## Shape (summary — see schema for the full spec)
```
scene_version "shadow-3d-scene-v1" · profile_id/version · bundle_id/hash · status_mode FIXTURE|LIVE|DEVICE
view_mode ready|case|council|decision|audit · layout_mode arc|dag|timeline|hybrid · layout_seed
nodes[]  { id, type, sequence, status VERIFIED|TAMPERED|NOT_VERIFIED|NOT_CHECKED|NOT_PRESENT|UNSUPPORTED,
           claim_ids/evidence_ids/source_ids, agent/tool/model, label_short/label_full, accessibility,
           label_priority P0..P3, focus_state/selected_state/expanded_state, layout_hint (advisory), evidence_ref }
edges[]  { id, type SUPPORTS|CONTRADICTS|DERIVED_FROM|APPLIES_POLICY|CHALLENGED_BY|SUPERSEDES|SEALED_BY, from, to, direction }
tamper   { failed_sequence, reason, downstream_sequences } | null
verification { record_integrity, signature, hash_chain, profile, source_resolution, external_anchor, analytical_correctness=NOT_EVALUATED }
```

## Invariants the contract test enforces
1. IDs unique; `sequence` dense 0..n-1; statuses in the enum.
2. Every edge references existing nodes.
3. The six independent checks are separate; **analytical correctness is NOT_EVALUATED** — never a green.
4. `tamper.downstream_sequences` == exactly the sequences after `failed_sequence`; the mutated node is
   `TAMPERED`, downstream nodes are `NOT_VERIFIED` (frozen).
5. Each profile has its own node types (banking source→…→audit_record; data-science dataset→feature→
   model→metric→experiment→…; coding issue→tool_call→diff→test→commit→…) — **not a recolored layout**.
6. Serialization is deterministic (attestable hash); no engine names appear in the scene.

## Adapter contract (both engines)
- Read the scene; build objects from IDs/types. Colour encodes **status only, never type** (matches the
  existing `demos/replay/3d/constants.js` principle). Status must also carry icon/shape + text (not colour
  alone). Positions come from `layout_hint` but may be recomputed per device — **semantics may not drift**.
- A cross-engine parity test (Phase 2) will assert node count / IDs / sequence / status / tamper point /
  source IDs / claim IDs / report hash are identical; only positions may differ.
