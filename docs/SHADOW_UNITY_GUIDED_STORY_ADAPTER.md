# Shadow Unity guided-story adapter

Native Unity rendering of a compiled guided story. No WebView / iframe, no XREAL SDK in this slice —
input arrives through `IShadowStoryInput` (a desktop keyboard mock here; a headset controller would
implement the same interface elsewhere). Positions are advisory and marked
`DEVICE VALIDATION PENDING` until measured on hardware.

## Files

Runtime (`Assets/ShadowLens/GuidedStory/`), all in the `ShadowLens` assembly:

- `ShadowGuidedStoryJson.cs` — a defensive mini JSON reader so Unity consumes the EXACT bytes of the
  parity anchor. Hard depth/byte caps; rejects `__proto__`/`prototype`/`constructor` keys.
- `ShadowGuidedStoryStatus.cs` — the vocabulary mirror (13 statuses, 15 trust dimensions) + status →
  severity / colour-key / shape. A parity test asserts this matches the Node vocabulary exactly.
- `ShadowGuidedStoryContract.cs` — typed model (`GuidedStorySemantic`, entities/relations/scenarios/
  steps). No positions — meaning only.
- `ShadowGuidedStoryLoader.cs` — fail-closed loader: rejects duplicate ids, dangling status/relation
  refs, unknown statuses/dimensions, undeclared dimension keys, and `ANALYTICAL_CORRECTNESS` ≠
  `NOT_EVALUATED` — the same contract as the Node compiler.
- `ShadowGuidedStoryUnityAdapter.cs` — pure `V3` layout (mirrors the Three.js layouts; arc reuses the
  tested `SpatialLayout.AuditArc`).
- `ShadowGuidedStoryState.cs` — step/scenario state machine (Next/Back/Restart never run off the ends).
- `ShadowGuidedStoryLocalization.cs` — EN/zh chrome strings.
- `IShadowStoryInput.cs` — input interface + `DesktopMockStoryInput` (no SDK reference).
- `ShadowGuidedStoryPlayer.cs` — the MonoBehaviour: spawns one primitive per node (shape by status),
  shows status as text too, dims out-of-focus nodes, steps via the input interface.

Editor (`Assets/ShadowLens/Editor/GuidedStory/`):

- `ShadowGuidedStoryImporter.cs` — `Shadow Lens → Validate Guided Story Snapshots` (runs the runtime
  loader so a bad snapshot is caught in the Editor).
- `ShadowGuidedStoryDemoSceneBuilder.cs` — `Shadow Lens → Guided Story Demo` builds a desktop scene
  with a camera, an honest banner (`FIXTURE · DESKTOP MOCK · DEVICE VALIDATION PENDING`), and the
  player wired to the three snapshots. Arrow keys step; L=language, F=2D, R=restart.

Data: `Assets/ShadowLens/GuidedStory/Snapshots/<id>.semantic.json` — byte-identical to the parity
anchors (regenerated by `fixtures/guided-stories/snapshots/build-snapshots.mjs`).

## Verification (Unity 6000.0.23f1, batch-mode `-nographics`)

- `ShadowLens.dll` compiles clean — 0 CS errors.
- EditMode `ShadowGuidedStoryEditTests`: **11/11 passed** (JSON parser incl. proto-key rejection,
  loader fail-closed incl. analytical-correctness guard, vocabulary mirror, all-layouts finite,
  seq-3 first-failure + downstream, focus+context, state machine bounds).
- PlayMode `ShadowGuidedStoryPlayModeTests`: **3/3 passed** (loads all 3 stories + spawns nodes,
  step/back/restart stay in bounds, switching stories rebuilds cleanly).

Result XMLs: `media/story-adapters/unity-tests/{editmode,playmode}-results.xml`.

## Not done in this slice

Android build (the stable APK is untouched) and on-device validation. Positions and legibility remain
hypotheses until measured on hardware — the demo banner and the tokens' `$note` fields say so.

## Reproduce

```
UNITY=/Applications/Unity/Hub/Editor/6000.0.23f1/Unity.app/Contents/MacOS/Unity
$UNITY -batchmode -nographics -projectPath apps/shadow-lens/unity \
  -runTests -testPlatform EditMode -testFilter "ShadowLens.Tests.ShadowGuidedStory*" \
  -testResults editmode.xml -logFile -
# (PlayMode: -testPlatform PlayMode -testFilter "ShadowLens.Tests.PlayMode.ShadowGuidedStory*")
```
Do not pass `-quit` with `-runTests` — it pre-empts the runner before tests execute.
