## The problem: rendering code owns business semantics (DUPLICATED across 3 places)
The same decision semantics — node sequence, the tamper failure point, status vocabulary, downstream
propagation — are **encoded independently in three renderers/data files**:

| Semantic | Three.js | Unity | Node core |
|---|---|---|---|
| tamper failure sequence | `constants.js MUTATE_SEQ = 6` **DUPLICATED** | `ShadowAuditChainData.BrokenAtSeq` **DUPLICATED** | `audit-ingested` / bundle prev_hash break |
| status vocabulary/colour | `constants.js STATUS` **DUPLICATED** | `ShadowDesignTokens` (Verified/Tampered) **DUPLICATED** | verify matrix statuses |
| provenance spine / node order | scene node build **DUPLICATED** | `ShadowAuditChainData.Chain` **DUPLICATED** | `claim-evidence-graph PROVENANCE_ORDER` |
| evidence bundle | inlined `demo-data.js` | `ShadowBankingNarrativeData` **DUPLICATED** | `fixtures/banking-*.mjs` |

→ Change the tamper point and you must edit it in ≥3 files; nothing guarantees Unity and Three.js tell
the same story. **This is exactly what the shared contract removes.**

## Target map
```
SHARED DOMAIN DATA  (shadow-3d-scene-v1 — engine-neutral: node/edge IDs, sequence, status, tamper seq,
                     downstream, labels, a11y, layout HINTS)
   ├─ UNITY ADAPTER   (reads the scene → builds GameObjects; positions may adjust for device)
   └─ THREE.JS ADAPTER(reads the same scene → builds Object3D; positions may adjust for viewport)
        → RENDERED OBJECTS   (positions differ; SEMANTICS + verification states must be identical)
```
Authoritative in the contract: semantic identity + verification/tamper states. Hints only: positions.
Never in the contract: GameObject/MonoBehaviour/Object3D names, engine materials, engine coordinates as
truth.
