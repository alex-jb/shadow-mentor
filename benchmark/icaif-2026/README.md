## What the numbers say

**Verdict accuracy 100% (200/200).** Gold labels are derived from the same
`LOAN_DEFAULTS` thresholds the council uses in `lib/run-loan-council.js`. The
100% figure is an internal-consistency verification of the deterministic
verdict resolver, not a novel accuracy claim. The paper should frame it that
way: "the deterministic verdict layer is coherent with its own labeling
policy; the LLM-rationale layer is the empirical anchor that generates
per-persona probability estimates and calibration curves in Section 5.3."

## Known schema gap the paper must address

`confidence` values in the current runLoanCouncil output are per-persona
constants (0.82 / 0.78 / 0.91 / 0.74 / 0.69). The paper Section 3.2 claims
"per-persona Brier decomposition" — that requires each persona to emit a
`P(class) ∈ [0, 1]` per event class, not a fixed confidence in the persona's
own verdict.

Two options for Section 5.3:

1. **Extend the schema** to emit `p_approve`, `p_escalate`, `p_block` per
   persona; refit temperature scaling per Guo et al. 2017; report reliability
   / resolution / uncertainty decomposition. Higher upside, ~1 week of
   engineering + LLM-rationale wiring cost.
2. **Reframe as weighted-verdict Brier** using the constant per-persona
   confidence as a persona-level weight (already shipped in
   `lib/confidence-weighted-verdict.js`). Cheaper. Loses the per-persona
   calibration curve claim; keeps the audit-primitive claim.
